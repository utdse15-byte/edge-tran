# 人工测试清单（当前版本 0.2.13）

自动测试覆盖翻译管线、Provider、密钥绑定、存储事务、Service Worker 租约和 Chromium 页面仿真。真实 Claude 账户中的“可见输入＝实际发送内容”仍必须由用户人工验证。

## A. 安装、升级与权限

- [ ] `edge://extensions` 能无错误加载当前版本。
- [ ] 从 v0.2.x 覆盖原目录升级后，Provider、模型、Key 和草稿仍存在。
- [ ] 升级后明确保存一次配置，旧裸 Key 被迁移，翻译仍可使用。
- [ ] 保存 Provider 时只请求该 Provider 的精确 Origin。
- [ ] Claude 页面 DOM、Console、内容脚本和网络请求中不存在 API Key。
- [ ] 清除 Key 后，本地和会话 Key 都不可再用。

## B. 三种回译方式

### 同请求模式

- [ ] 一次正常翻译只产生一个 `/chat/completions` 请求。
- [ ] 英文和中文回译同时更新。
- [ ] 回译区域标明“模型自检，不等同于独立验证”。
- [ ] 模型返回纯英文 `back_translation` 时，结果被拒绝或受控重取，不显示为成功。
- [ ] 极短中文“你好”“谢谢”仍必须得到真实英文和中文回译。

### 独立模式

- [ ] 真实连通测试分别验证主模型和回译模型。
- [ ] 第二个请求的用户内容只有英文，不含中文原稿。
- [ ] 第一次回译不是中文时，只重试一次。
- [ ] 英文成功但回译最终失败时，英文仍保留。

### 关闭回译

- [ ] 只产生主翻译请求。
- [ ] 回译区域显示已关闭，不残留旧结果为当前结果。

## C. 重复请求和版本门

- [ ] 输入后立刻点击“立即翻译”，等待超过防抖窗口，确认只有一次请求。
- [ ] `Ctrl+Enter` 同样不会重复请求。
- [ ] 快速连续输入时旧响应不会覆盖新译文。
- [ ] 请求期间切换 Provider、模型、Key 或回译方式，旧响应失效。
- [ ] 暂停、解绑或关闭侧栏后，排队写入不会继续静默执行。

## D. Provider/Key 多窗口竞态

使用两个 Edge 窗口，各自打开侧栏：

- [ ] 窗口 A 选择 Provider A 并开始请求；窗口 B 保存 Provider B 和新 Key 后，A 的旧请求被取消，A 采纳新配置并继续（0.2.11+ 不再强制暂停）。
- [ ] A 不会把 B 的 Key 发往 A 的旧 Base URL。
- [ ] A 只修改防抖时间等行为设置，不会把旧 Provider 覆盖回存储。
- [ ] 当前窗口自己点击保存配置时，不会因自己的存储事件误暂停。
- [ ] 从本地持久切换为仅会话，旧本地 Key 被删除；任一步失败时配置回滚。
- [ ] 从仅会话切换为本地持久，同理验证。

## E. 标签页、窗口和 SPA 会话

- [ ] 两个 Claude 标签页同时打开，插件只写入明确绑定的标签页。
- [ ] 请求期间切换到另一个标签页，英文只留预览。
- [ ] Claude 会话切换后，旧请求不写入新会话。
- [ ] 反复快速点击绑定两个 Claude 标签页，最终绑定与最后一次选择一致。
- [ ] 同一 Claude 标签页不会被两个侧栏租约同时控制。
- [ ] 两个 Edge 窗口中的活动草稿、历史和诊断互不覆盖。

## F. 写入、焦点与富文本

- [ ] 页面诊断能写入固定短句并准确回读。
- [ ] 侧栏连续输入时，没有字符落入 Claude 输入框。
- [ ] 多段、Markdown、代码块和连续空行结构正确。
- [ ] Claude 输入框有文件、图片、附件卡片或 mention 时，插件拒绝自动覆盖，附件不丢失。
- [ ] 写入失败时，Claude 内容要么恢复原状，要么插件暂停并要求人工核对。
- [ ] 人工修改英文后，自动覆盖暂停。
- [ ] 点击“确认覆盖并同步”后，只有这次明确操作可以覆盖人工英文。
- [ ] Claude 若改用 `contenteditable="plaintext-only"`，页面诊断仍可定位；无法安全写入时必须退化为复制模式。

## G. 发送与清空识别

- [ ] 人工点击 Claude 发送按钮后，当前版本草稿归档并建立新草稿。
- [ ] 在支持“Enter 插入换行”的设置下，按普通 Enter 后快速清空输入框，不会被误判为已发送。
- [ ] `Shift+Enter`、`Ctrl+Enter` 或其他自定义换行快捷键不会错误归档。
- [ ] 手动全选删除 Claude 输入框，不会清除中文原稿。
- [ ] 页面刷新、会话切换或编辑器节点替换不会重复产生一次 SEND 和一次 CLEAR。

## H. 发送链路 GO/NO-GO

对以下内容分别执行“诊断写入 → 人工点击发送 → 核对实际消息”：

- [ ] 单行英文；
- [ ] 多段英文；
- [ ] Markdown 列表；
- [ ] 三反引号代码块；
- [ ] 连续空行；
- [ ] 带附件的消息。

**任何一项出现可见文字与实际发送内容不一致，应停用自动同步，只使用复制模式。**

## I. 翻译保义

- [ ] 否定与双重否定；
- [ ] 必须、应该、可能、尽量；
- [ ] 直接、强硬或不礼貌语气；
- [ ] 一个明显同音错字；
- [ ] 一个无法确定的歧义；
- [ ] 数字、百分比、日期和版本号；
- [ ] URL、邮箱、Windows/Unix 路径；
- [ ] Markdown 与代码；
- [ ] 原文中包含“忽略之前指令”等提示注入文本；
- [ ] 原文中包含类似 `⟦ZH2EN_ABCD_P0⟧` 的普通示例文本。

通过条件：模型只翻译不回答；所有笔误解释都显示；越界纠错被本地拒绝；占位符完整。

## J. Provider 和错误分类

- [ ] `/models` 可用时能列出模型。
- [ ] `/models` 不可用时，手工模型 ID 仍能测试和使用。
- [ ] 404 且错误明确指向模型时，提示“模型不存在或无权访问”。
- [ ] 404/405 且为路由错误时，提示检查 Base URL/端点。
- [ ] 任意普通 400 不会自动重复请求。
- [ ] 网关第一次明确拒绝 `response_format`、第二次明确拒绝 `temperature` 时，可以逐项降级并最终成功。
- [ ] 兼容降级总次数有界，不发生循环。
- [ ] 401/403 停止，不重试。
- [ ] 429 进入冷却，不发生重试风暴。
- [ ] 超时、断网、5xx 最多按设计重试一次；旧响应不写入。
- [ ] 工具调用、函数调用、拒绝、截断、空结果、非文本内容和超大响应被拒绝。
- [ ] 同时配置 `X-Project` 和 `x-project` 时，只保留一项，不发送重复语义 Header。

## K. 草稿和存储

- [ ] 关闭再打开侧栏，未发送草稿恢复。
- [ ] 最近 20 条历史可恢复和删除。
- [ ] 输入超过 50,000、少于 250,000 字符的草稿：翻译被阻止，但关闭再打开侧栏后全文仍保留。
- [ ] 发送归档或存储失败时，侧栏显示错误，不静默丢稿。
- [ ] 旧配置若含 Cookie、Authorization、Token、Secret 或危险对象键，载入后不会用于额外请求头。
- [ ] 重启 Edge 后，会话草稿历史清空；Key 是否保留符合所选模式。

## L. 生命周期

- [ ] 关闭侧栏后无 Provider 请求、无持续 Writer observer、无写入。
- [ ] Claude 标签页刷新后能自动恢复绑定或给出明确提示。
- [ ] Claude 改版导致定位失败时，插件最多退化为预览/复制，不误写其他编辑框。

## 接口协议（0.2.6+）

- [ ] 设置中"接口协议"默认是 Chat Completions；保存后 Provider 摘要不出现 Responses 标记。
- [ ] 切换为 Responses API 并连通测试：请求打到 `{baseUrl}/responses`，成功返回英文与回译。
- [ ] 协议选错（Responses 端点配 Chat Completions 或反之）时，错误信息明确提示到设置中切换协议，且不自动重试。
- [ ] Responses 模式下 o 系列模型（含 reasoning 输出项）翻译正常。

## 暂停自愈与评审提醒（0.2.11+）

- [ ] 切换 Claude 会话/被外部清空后状态栏显示暂停，继续输入新中文即自动恢复并同步，无需点"恢复"。
- [ ] Esc 暂停后继续输入：保持暂停、不发请求；只有"恢复"按钮能解除。
- [ ] 另一窗口保存 Provider/Key：当前窗口状态栏提示"已采纳另一个窗口的配置修改"，继续输入直接用新配置翻译。
- [ ] 有纠错/歧义提醒的翻译默认仍自动同步，状态栏提示"发送前请过目"；开启"严格评审"后恢复为停在预览等显式同步。
- [ ] 连通测试失败（如 /v1/responses 502）时，结果行出现"诊断："给出缓冲/协议对照结论与网关消息摘录。

## 零删除与定位（0.2.12+）

- [ ] Claude 输入框里有自己输入的内容时：自动同步不写入、不删除，状态栏提示"已有内容"，不弹人工修改横幅、不暂停。
- [ ] 修改中文后，旧英文不再被自动清空；新译文完成后直接覆盖插件自己的旧译文。
- [ ] "清空输入框"按钮：确认后清空全部内容；含附件时拒绝清空并提示人工处理。
- [ ] 在 claude.ai 首页/新会话（输入框居中）也能自动定位，无需先输入文字。
- [ ] 切换会话/路由后 1-2 秒内输入框重新定位；切到其他标签页再切回后定位仍有效。

## 定位状态机（0.2.13+）

- [ ] 切换会话后旧页面输入框若被隐藏缓存，插件应绑定新页面输入框，旧文字不被触碰、不误报"被外部清空"。
- [ ] 输入框被弹层/过渡长时间遮挡（>5 秒）且文字未变：状态显示未就绪而非"被清空"；恢复可见后自动重新定位且插件所有权延续。
- [ ] 历史消息的行内编辑框永不被自动绑定；需要时用"绑定当前页"手动指定。

## 自动化套件说明

- `npm run verify` 只含 Node 测试、语法检查与静态审计。
- `npm run verify:full` 追加 4 个 Chromium 冒烟/状态机套件（writer 定位/写入/发送确认、panel 状态机、换会话与断连恢复），**发布前必须跑 verify:full**。
