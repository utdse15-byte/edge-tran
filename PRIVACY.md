# 隐私与安全边界

## 扩展会处理什么

扩展侧边栏会保存或处理：

- 你主动输入的中文草稿；
- Provider 配置、模型 ID、回译方式和行为设置；
- 英文译文、中文回译、笔误、歧义和本地校验警告；
- 当前 Edge 会话内按窗口隔离的活动草稿、最近 20 条历史和不含正文的诊断事件；
- API Key（本地持久或仅会话，由你选择）。

活动草稿最多保存 250,000 字符；翻译请求仍限制为 50,000 字符。历史总量另有本地上限。

## 哪些内容会发给 Provider

### 同请求模式（默认）

一个主请求包含：

- 当前中文草稿；
- 翻译和回译规则；
- 受保护词及随机占位符。

模型在同一个响应中返回英文和 `back_translation`。同一模型看到了中文原稿，因此这是快速自检，不是独立验证。

### 独立回译模式

第一个请求只要求英文。第二个请求只包含：

- 已生成的英文；
- 英文→简体中文规则；
- 从英文中派生的代码、URL、路径和保护词占位符。

第二个请求不包含原始中文。

### 关闭回译

只发送中文主翻译请求。

## 不会读取或发送什么

- Claude 的回复、历史消息或其他对话正文；
- Claude Cookie、账户资料或站点内部网络请求；
- 页面标题、完整会话 URL、其他标签页正文或浏览历史；
- 自动遥测、使用统计或崩溃上报；
- API Key 到 Claude 页面或内容脚本。

## API Key 存储与 Provider 绑定

插件启动时强制将本地和会话扩展存储限制为 `TRUSTED_CONTEXTS`。注入 `claude.ai` 的内容脚本不读取存储，也没有任何 Provider 网络代码。

v0.2.2 将 Key 保存为版本化记录，包含：

- Key 本身；
- `credentialId` 凭据代次；
- Provider 绑定摘要；
- 更新时间。

Provider 绑定由以下非秘密配置规范化生成：

- Base URL；
- 鉴权 Header；
- 鉴权前缀；
- 额外路由 Header；
- Key 保存模式和当前 Provider 凭据代次。

每次请求前，扩展会重新读取当前存储 Provider 与 Key 记录并核对。旧窗口缓存的 Provider 不能读取另一个窗口刚保存的新代次 Key，也不能把该 Key 发往旧目标。

保存配置时以下步骤作为一个事务：

1. 生成新的凭据代次；
2. 写入新 Key 到所选存储区域并绑定 Provider；
3. 删除另一存储区域中的旧 Key；
4. 保存 Provider 和普通设置。

失败时尝试恢复旧 Secret、Provider 和设置。

v0.2.1 及更早版本的裸字符串 Key 只在旧 Provider 尚无凭据代次时兼容读取。用户下一次明确保存配置后，它会迁移为绑定记录。

扩展不向运行时功能提供“只按存储模式读取 Key”的通用接口；翻译、模型检测和连接测试都必须携带当前 Provider，并通过绑定校验。

## 多窗口配置变化

Provider、设置或 Key 在另一个 Edge 窗口被修改时：

- 当前窗口取消在途翻译和回译；
- 旧结果和排队写入失效；
- 自动流程暂停；
- 用户需要检查新配置后恢复。

当前窗口自己的事务保存会被识别为自身写入，不会因存储事件回声误暂停。

## Header 与网络边界

额外请求头不会保存 Cookie、Authorization、API Key、Token、Secret、内容类型、Host、Origin、Referer、`Sec-*`、`Proxy-*` 或危险对象键。升级前残留的这些字段也会在载入时清理。

Header 名称按大小写不敏感去重。Provider 请求使用精确获授权 Origin；不做跨 Provider 自动故障转移，不静默把同一文本发送给另一家供应商。

本地持久模式是个人便利取舍，不是系统密钥库。它不能抵御已经以你的 Windows 用户身份运行、能够读取浏览器资料目录或记录键盘的恶意软件。建议使用独立、可撤销、可限额的 Key。

## Claude 页面行为

插件只会：

- 定位明确绑定标签页中的 Claude 消息输入框；
- 在目标标签页处于前台时，低频、整段写入英文并回读；
- 观察该输入框是否被人工修改、清空或替换；
- 记录可信发送按钮/表单/键盘意图，用于区分发送后清空和普通清空；
- 在存在附件或不可安全替换的富文本节点时拒绝自动覆盖。

插件不会自动发送、抓取回复、调用 Claude 内部接口、拦截 Claude 网络、读取 Cookie、伪造设备指纹或规避检测。
