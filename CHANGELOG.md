# Changelog

## 0.2.7 — 2026-07-20

外部深度审计核验修复版:对第三方 AI 审计的 33 项发现逐条对照代码验证。31 项落实修复(其中数量保真、复述判定、路径识别 3 项为部分修复并文档化剩余边界);2 项(中文数词短语的语义判定、旧草稿跨窗口迁移原子性)接受为文档化已知边界;个别子例经验证为已有防护覆盖。

### 安全与状态机（panel.js / sw.js / lib/storage.js）

- 【高】暂停原因隔离:所有非目标丢失的暂停点(共 14 处)显式清除 `pausedByTargetLoss`,自动重绑成功不再可能误清外部配置审查、人工修改等其他暂停。
- 【高】审阅门禁:纠错、歧义或数量/复述/回答式/语序类警告存在时不再自动同步;英文保留在预览,用户点击"同步到 Claude"即显式批准。独立回译照常调度。
- 【高】SW 写命令在前台检查 `await` 前快照 lease/boundTabId/bindGeneration,恢复后全部一致才放行,并以快照 lease 发送;等待期间解绑+重绑不再让旧命令借新租约执行。
- 【高】配置保存回滚前,每个密钥槽位校验当前记录的凭据代次属于本事务/快照/为空,另一窗口保存中途写入的新 Key 不再被回滚覆盖(不可判定时失败关闭)。

### 翻译校验（lib/validation.js / lib/translator.js）

- 【高】纠错拒绝扩展:极性/方向/状态单字(是否开关多少前后真假对错增减升降高低上下左右内外正反新旧好坏快慢)与大写/异体数字(壹贰叁肆伍陆柒捌玖拾佰仟廿卅);超过 8 条的纠错列表整体越界;歧义超过 6 条产生显式警告。
- 【高】未翻译中文残留 ≥60% 从警告升级为硬失败("你好 a"不再通过);回译中文占比过低(≥8 字原稿 <20%)硬失败;"x中"式回译不再有效。
- 【高】中文数词数值核对:解析两字以上中文基数词(三十、一千二百、五万),与译文中的阿拉伯/中文数字数值比对,不一致产生数量警告并触发审阅门禁。
- 【高】占位符还原后英文/回译超过 100,000 字符硬失败,不再触发存储静默截断导致的状态失配。
- 双向覆盖/隔离符与行间注控制字符(U+202A-202E、U+2066-2069、U+FFF9-FFFB)一律硬拒绝。

### Provider 协议（lib/provider.js）

- Responses 解析严格化:对象标记存在即必须为 `response`;顶层仅 `completed`(或缺省)算成功,`in_progress/queued` 判未完成;消息项要求 assistant 角色与 completed 状态。
- Chat Completions:`finish_reason` 白名单(stop/end_turn/eos/缺省),非 assistant 角色拒绝,移除 legacy `choices[0].text` 回退。
- 兼容降级参数别名移除裸 `text`,无关 400 不再触发去 JSON 重试;404/405 文案协议中立。
- `/models` 返回非模型列表结构显式报 `unsupported_response_schema`,不再显示为"没有模型"。
- 密钥样式请求头拦截扩展(access-key、credential、signature、bearer、session-id、独立 key/auth/sign 词段),storage 侧同步;自定义鉴权 Header 槽位不受影响。
- 诊断端点路径改为默认脱敏:仅保留常见路由词、版本号与短纯字母段。

### Writer（writer.js）

- 写入/清除类命令执行前重新运行完整可编辑判定,隐藏/禁用/只读控件返回 `composer_not_editable`。
- 周期扫描在编辑器仍有效时也调度重排名,属性级 hydration 出现的更优主编辑器可被接管(+60 滞后不变)。
- 附件失败关闭覆盖仅带 aria-label 附件语义的兄弟节点芯片;按钮等交互控件排除,工具栏"添加附件"不会误触发。

### 占位符保护（lib/placeholders.js）

- Markdown 围栏改为逐字符扫描:开栏 3+ 个反引号/波浪线,闭栏必须不短于开栏(````…````不再被内部 ``` 提前闭合);行内代码支持等长多反引号(``a`b``)。
- Unix 路径识别扩展分隔上下文(：、=、[、,、;等)并支持 `~/`、`$VAR/` 前缀。
- 保护词先去重/去空再取 500 上限;逐码元推进以覆盖重叠出现。

### 文档与流程

- README 安装/升级说明去除固定 v0.2.2 目录名;BUILD_REPORT、TESTING 更新到当前版本并覆盖接口协议与浏览器套件说明。

### 测试

## 0.2.6 — 2026-07-20

Responses API 支持与使用稳定性修复版（针对用户反馈的四类问题）。

### OpenAI Responses API（/v1/responses）

- 设置新增"接口协议"选项：Chat Completions（默认）或 Responses API。协议是显式选择,插件仍不自动猜路由、不跨协议重试。
- Responses 请求：system 提示词映射到 `instructions`,用户内容作为 `input_text` 输入项,`store: false` 退出服务端响应留存;JSON 模式使用 `text.format`。
- Responses 解析:遍历 `output` 数组(忽略 o 系列 `reasoning` 项),`output_text` 拼接;`refusal` 部件 → 拒绝,`incomplete/max_output_tokens` → 截断,`function_call` 等工具项 → 硬拒绝;`failed` 状态归类为逻辑错误。
- 协议错配双向可判定:Responses 模式收到 Chat Completions 结构(或反之)会提示到设置中切换协议。
- 兼容降级把 `text.format` 参数拒绝映射到统一的 JSON 模式能力位,`temperature` 独立降级;能力签名纳入协议维度,切换协议不会误用另一协议学到的降级。
- Base URL 规范化支持粘贴完整 `/responses` 地址;裸 `/responses` 收到 HTML 时同样提示可能缺少 API 前缀。

### 使用稳定性（用户反馈修复）

- 【回归修复】"立即翻译"按钮不再可能永久禁用:0.2.5 给按钮恢复加身份守卫后,任何经 `abortInFlight` 的中止(暂停、换绑、清空草稿、SW 重启)都会跳过恢复;现在中止即恢复按钮。
- 空输入框定位:新增 1 秒级低频重扫。Claude 空态编辑器经属性级 hydration 变为可定位时(childList 观察器看不到),不再需要用户先往页面输入内容。
- 普通换会话不再强制暂停:仅当插件拥有的文本被换会话孤立时才暂停;在途翻译不再被中止(写入侧本就有身份校验,中止只会浪费已计费请求),英文预览保留并提示可手动同步。
- 可恢复的目标丢失(页面刷新、MV3 服务工作线程重启)自动重绑成功后自动恢复运行,不再每次都要人工点"恢复";用户主动暂停不受影响,仍需手动恢复。

### 测试

- Node 测试 62 → 66 项:Responses 请求构建与解析、text.format 降级、拒绝/截断/工具调用/协议错配、存储协议字段规范化与 URL 后缀剥离。
- Writer Chromium 冒烟新增属性级 hydration 定位场景;Panel 状态机新增换会话不暂停、中止后按钮恢复、目标丢失自动恢复三个场景。

## 0.2.5 — 2026-07-20

深度缺陷审查修复版（四路并行审查 + 逐项人工验证，共修复 1 高危 / 5 中危 / 7 低危）。

### 翻译校验（lib/validation.js）

- 【高危】拒绝涉及中文数字与序数的"明显错字"纠错（如 三十→五十、第三→第五）。此前中文数字是普通汉字码点，能通过短片段形状校验，且 ASCII 数字软警告也不会触发，模型声称的纠错可以静默改变数量后直接同步进 Claude。
- 修复"回答式措辞"软警告的正则：`sure[,!]` 后误置的 `\b` 要求标点后紧跟单词字符，导致 "Sure, …"/"Sure!" 永远不匹配；否定词黑名单补充"莫"。

### 发送确认与绑定（writer.js）

- 真实发送导致编辑器被卸载、替换节点尚未挂载时，受信的 Enter 发送意图不再因 targetEpoch 递增而失效：发送仍被确认归档，而不是降级为伪 TARGET_CLEARED（草稿不归档、面板误暂停）。
- 普通 Enter 只在文档末尾插入换行（尾部代码块/列表,ProseMirror 在 keydown 阶段吞掉原生 input 事件）时,通过原始序列化对比取消发送意图；随后的清空/切换会话不再被伪造成已发送并清空中文草稿。
- attach 时对齐 lastUrl 并静默轮换会话：解绑期间发生的 SPA 导航不再让绑定后的第一个写入命令必然失败于 writer_session_changed 并强制暂停。
- 手动绑定切换到另一个空编辑器、旧编辑器文本原封未动时,不再伪造"输入框被外部清空"事件与暂停。

### 服务工作线程（sw.js）

- 换绑失败（not_claude / tab_unavailable）时释放旧标签页的租约与所有权。此前面板已按"完全解绑"处理（tabId 置空,幽灵事件过滤器失效）,而 SW 仍把旧 writer 的 SEND_CONFIRMED 等事件转发给面板,可触发错误的草稿归档。

### 侧栏面板（panel.js）

- 暂停→恢复（Esc、暂停按钮、自动开关）与取消的 IME 组合不再对"未变化且已翻译"的草稿重复发起付费请求,也不再用新生成的英文覆盖已核对、已同步的英文。
- 被 Ctrl+Enter 取代的旧翻译请求不再提前恢复"立即翻译"按钮（重复请求锁在新请求在途期间失效）；"缺少主翻译模型"提前返回路径补充按钮恢复。
- 诊断写入/清除接入目标状态机：写入成功后不再保留绿色"已同步"徽章与人工修改横幅（横幅上的"作为核对基线"会把诊断短句当作人工英文采纳）,清除成功后徽章复位。
- 旧存储中早于当前校验规则的 Base URL（如带 query 参数）不再让"保存设置"永久失败：无法规范化的旧 URL 视为无旧权限模式,不再阻塞撤销/保存流程。
- 运行时学习的兼容能力降级不再被普通设置保存持久化：保存只写入存储中已有的能力快照,显式连通测试结果仍可覆盖——与 0.2.2 文档承诺一致。

### Provider（lib/provider.js）

- claude.ai 禁用检查覆盖 FQDN 尾点形式：`https://claude.ai./v1` 不再绕过"Claude 站点不能作为 Provider"的硬禁止。

### 测试

- Node 测试扩充到 62 项：中文数字纠错拒绝、Sure 开头回答式警告、尾点 claude.ai 拒绝、换绑失败释放租约与幽灵事件阻断。
- Writer Chromium 冒烟新增 4 个回归：无替换卸载后的发送确认、尾部换行意图取消、解绑期导航后的静默会话轮换、手动换绑不伪造外部清空。已验证新场景在修复前代码上确实失败。

## 0.2.4 — 2026-07-19

针对 Provider 伪成功响应与 Claude 编辑器定位/手动绑定的热修复版。

### Provider 协议诊断

- HTTP 200 HTML、空正文、非 JSON、200 逻辑错误、Responses API、未知 JSON、空 `choices` 与空 assistant 内容分别分类，不再统一显示 `empty_response`。
- 设置页“检测模型”和“真实连通测试”直接显示安全协议摘要，包括 HTTP 状态、Content-Type、响应大小、请求 ID、端点和顶层结构。
- 裸 `/chat/completions` 或 `/models` 返回 HTML 时给出“可能缺少 API 前缀”的诊断提示，但不自动尝试 `/v1` 或重复请求。
- 恢复 `redirect: "error"`，并使用内部 tagged result，远端 JSON 无法伪造本地协议错误通道。
- 诊断元数据经过白名单与脱敏，不保存 API Key、Authorization、提示词、请求正文、完整 HTML 或完整远端错误内容。

### Claude 输入框与手动绑定

- 输入语义扩展到父级 `aria-label`、`aria-placeholder`、`data-placeholder`、`data-testid` 等属性。
- 空输入状态下 disabled 的真实发送按钮仍作为定位证据；隐式 submit 工具按钮不会被误当作发送按钮。
- 支持短内层 Lexical/ProseMirror/plaintext-only 编辑节点，并在后续 hydration 出现更优候选时重新排名。
- 历史消息行内编辑器降权；已验证的手动目标不会被普通自动重排抢占。
- 手动绑定使用 `composedPath()`，等待 React/Lexical 重渲染并验证稳定性；页面导航、租约变化、detach 或新请求会取消旧的异步验证。
- 同一节点再次手动绑定也会刷新 Writer 状态，成功结果必须携带 `composerReady`。

### 测试

- Node 测试扩充到 58 项，覆盖 HTML/空正文/非 JSON/结构不兼容、诊断脱敏、远端 marker 注入和禁止重定向。
- Writer Chromium 冒烟新增父级语义 + disabled 发送按钮、React 节点替换和绑定期间 detach 取消回归。

## 0.2.3 — 2026-07-19

深度缺陷审查修复版。完整问题清单与验证状态见 [BUGCHECK_REPORT.md](BUGCHECK_REPORT.md)。

### 发送确认与清除抑制（writer.js）

- 修复自动清除失败后遗留的写入抑制状态可能无限期吞掉真实发送的问题：失败路径恢复或解除抑制，发送意图只在 DOM 真正被本插件清空后才作废；非受信回声抑制不再匹配空文本。
- SPA 导航后保留未变化的插件草稿所有权：`previouslyOwned` 重认领比较改用导航前的 `lastWritten`，同节点分支同样恢复所有权,不再把插件文本误判为人工内容。
- 普通 Enter 之后的受信删除/撤销输入会取消键盘发送意图,防止“无效 Enter + 3 秒内手动清空”被伪造成已发送并归档。
- 输入框被覆盖层短暂隐藏时保留绑定并宽限重查（上限 5 秒），不再把仍然存在的文本误报为外部清空。
- 写入命令在“发送清空后、reconcile 前”到达时改走 `confirmClear`,由发送意图决定归档,不再以空文本报人工修改。
- 发送按钮判定不再把表单内隐式 `type="submit"` 的工具按钮当作发送控件。
- 发送意图 keydown 监听移到 document 捕获阶段,先于页面编辑器自身的 Enter 处理执行。
- 文本控件（textarea/input）目标增加轮询,页面程序化清空同样能确认发送。
- 附件失败关闭检测扩展到编辑器容器内的兄弟节点（仅识别明确的附件标记属性）。
- 预渲染文档推迟到激活后才连接写入端口。

### 服务工作线程（sw.js）

- 新增 `tabs.onAttached`：绑定的标签页拖到其他 Edge 窗口后写入不再永久失败于“不在前台”。
- 新增 `tabs.onReplaced`：标签页被浏览器替换时清除失效绑定并通知面板。
- 拒绝 `documentLifecycle` 非 `active` 的写入端口连接,预渲染文档不能抢占 writer 槽位或继承租约。
- 面板端口注册校验发送方：来自标签页上下文的 `zh2en-panel` 连接一律拒绝。
- 初始化失败不再留下未处理拒绝,工具栏按钮仍可打开侧栏展示面板自身的失败关闭错误。

### 侧栏面板（panel.js）

- 暂停或人工修改状态下的显式同步不再被静默丢弃,状态栏明确提示；关闭回译模式下“正在准备同步…”不再卡死。
- 重新绑定不再无条件把目标标记为“已同步”：仅当 Claude 中内容与当前中文修订的英文一致才显示绿色徽章,过期内容恢复旧译文警示并重排自动清除。
- 目标标签页关闭/被接管时重置 `pluginOwned`/`composerReady`,消除“幽灵所有权”导致的错误横幅。
- 设置对话框已打开时的重复 `openSettings()` 变为幂等,不再重置表单、吞掉在途检测请求或抛 `InvalidStateError`。
- “立即翻译”直接中止独立回译时同步重置回译阶段,徽章不再卡在“等待回译”。

### Provider 与存储（lib/）

- Base URL 校验拒绝空查询/空片段分隔符（`…/v1?`、`…/v1#`）,此前会生成永远 404 的端点。
- Header 值、鉴权前缀与 API Key 校验扩展到拒绝非 Latin-1 字符,避免 `fetch()` 抛错被误诊为“网络错误”。
- 超过 128KB 的错误响应体按 HTTP 状态分类（保留 429 的 `Retry-After` 与 401 的提示）,不再被 `response_too_large` 掩盖。
- 凭据绑定串的 Header 排序改为码元序,不再受语言环境/ICU 版本影响。
- 凭据绑定串超限时在保存阶段显式失败,不再出现“保存成功但 Key 永远读不出”。
- 配置保存回滚前校验当前存储归属,不再覆盖另一窗口已提交的保存。
- `minimum_chrome_version` 提升到 118（`storage.local.setAccessLevel` 的最低要求）。

### 测试

- Node 测试扩充到 57 项：新增 URL 分隔符、非 Latin-1 Header、大错误体分类、绑定排序确定性、预渲染拒绝、跨窗口移动标签页与标签页替换回归。
- Chromium 冒烟与状态机安全测试全部通过。

## 0.2.2 — 2026-07-19

### Provider、Key 与多窗口一致性

- API Key 改为带版本的 Secret envelope，并与规范化后的 Provider 目标及 `credentialId` 强绑定。
- 请求读取 Key 时同时验证当前存储 Provider、Base URL、鉴权 Header、鉴权前缀、路由 Header、Key 存储模式和凭据代次。
- 修复窗口 A 缓存旧 Provider、窗口 B 保存新 Key 后，窗口 A 可能把新 Key 配到旧请求目标上的竞态。
- 旧版裸字符串 Key 仅在 Provider 仍为旧格式时兼容读取；下一次明确保存自动迁移为绑定记录。
- 删除运行时无 Provider 绑定的 Secret 读取/写入导出，翻译、模型检测和连接测试必须使用绑定读取。
- 另一个窗口改变 Provider、模型、Key 或保存方式时，当前窗口取消请求、作废运行上下文并自动暂停。
- 当前窗口自身保存产生的存储事件通过精确签名和凭据代次识别，不会误触发外部修改警告。
- 行为设置保存不再连带覆盖 Provider；运行时能力降级默认只更新当前上下文。

### Provider 兼容与请求边界

- 自定义 Header 名称按大小写不敏感去重，避免重复语义 Header。
- 若网关依次拒绝 `response_format` 和 `temperature`，允许最多两次、逐项且有界的兼容降级。
- 任意无关 400/422、模型错误或业务错误不触发兼容重试。
- 保留工具调用、函数调用、结构化拒绝、非文本内容、截断、过滤、空响应和超限响应的硬拒绝。

### 翻译与回译校验

- 用户原文中类似插件占位符的普通文本不再触发未知令牌硬失败。
- 短中文必须产生真实英文；只有原文为中文数词时才接受纯数字结果。
- 同请求回译不含中文时改为硬失败；独立回译可受控重试一次。
- `corrections` 和 `ambiguous` 非数组时拒绝结果。
- 修复随机占位符中的数字造成错误数字差异警告的问题。
- 保护内容恢复后再执行回译语言和数字检查。
- 独立回译同样保护 URL、代码、路径和保护词，且请求中仍不包含中文原稿。

### Provider 与配置

- 只有明确不支持 `response_format` 或 `temperature` 时才降级一次；任意 400 不再重复请求。
- HTTP 404 区分模型不存在/无权限与端点路径错误。
- 拒绝结构化 refusal、工具/函数调用、非文本内容 part、截断、过滤和异常大响应。
- 载入旧 Provider 设置时清除 Cookie、Authorization、Token、Secret、内容控制头和危险对象键。
- 无效鉴权 Header 或含控制字符的前缀在运行前回退到安全默认值。
- Provider、设置、Secret、草稿、历史和诊断的存储结构统一限长、限量和类型规范化。

### 草稿与错误恢复

- 发送归档、不确定发送归档和清空草稿的异步存储失败现在会记录并提示，不再产生未处理 Promise rejection。
- 活动中文草稿存储上限提高到 250,000 字符；翻译上限仍为 50,000，避免超长未译草稿关闭侧栏后被截断。
- 清理 Claude 失败后，侧栏可以清空自身草稿，但保持自动同步暂停并明确提示 Claude 仍有内容。
