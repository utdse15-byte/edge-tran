# Claude 中文→英文输入助手（Edge，个人侧载版）

在 Edge 侧边栏输入中文，插件使用你自己的 OpenAI、xAI/Grok 或 OpenAI Chat Completions 兼容供应商生成英文，并把完整英文同步到 `claude.ai` 的消息输入框；中文回译用于检查语义。

当前版本：**0.2.12**。

## 0.2.12 更新重点（零删除与定位稳定）

- **自动流程永不删除**:移除"旧译文自动清理";插件旧译文由新译文直接覆盖,你自己放进输入框的内容自动流程绝不触碰。删除只有一条路:新增的**"清空输入框"按钮**(需确认,附件受保护)。
- **已有内容安静处理**:绑定时输入框里有你的内容,不再弹横幅、不暂停;新译文停在预览,点"同步到 Claude(覆盖)"或"清空输入框"由你决定。
- **首页/新会话居中输入框可定位**:修复切页后定位不上的根因(评分门槛只认底部输入框);另补后台切回、状态询问时的重定位触发。

## 0.2.11 更新重点（摩擦清除与可诊断化）

- **系统性暂停全部自愈**:会话切换、输入框被外部清空、不确定发送、恢复历史草稿、保留人工英文之后——继续输入新中文或重新定位成功即自动恢复,不再要求点"恢复"。Esc/暂停按钮的用户暂停语义不变(输入不会解除)。
- **另一窗口改配置不再暂停**:自动采纳新配置继续运行(在途请求照旧作废、凭据代次校验不变)。
- **评审提醒默认不再拦截同步**:纠错/歧义/数量提醒醒目展示但英文照常写入 Claude 输入框(发送仍由你);设置里可开"严格评审"恢复旧行为。硬校验失败仍直接报错。
- **/v1/responses 等接口故障可诊断**:错误现在带网关原始消息摘录(脱敏)、request-id、协议与流式标记;连通测试镜像流式设置,失败时自动做"缓冲 vs 流式""responses vs chat"对照定位,最多 2 次小请求。

## 0.2.10 更新重点（体验收敛）

- 主按钮改为**"翻译并同步"**——按钮文案如实覆盖副作用(请求模型 + 写入 Claude)。
- 自动化改为**三态**:手动 / 自动翻译(不写入 Claude)/ 自动翻译并同步;旧"自动"开关自动迁移。
- **Esc 只暂停不恢复**:紧急按键只能降低自动化,恢复需点"恢复"按钮。
- 人工修改冲突卡默认**保护你的修改**:"保留 Claude 当前英文"为主操作,"重新翻译并覆盖…"需二次确认。
- 回译选项标注请求次数;字数统计显示"n / 50,000 可翻译"双上限;补全键盘可见焦点。

## 0.2.9 更新重点（思考强度兼容层）

- 设置新增**思考策略**(继承默认/关闭/手动档位)与**思考参数方言**(OpenAI Chat、OpenAI Responses、OpenRouter、DeepSeek、通用 thinking.type、通用 enable_thinking、不发送)。方言独立于接口协议,由你的接口/反代文档决定。
- 省略≠关闭≠最低档:三种意图分别编码;未知反代默认**不发送**任何思考字段;老配置行为零变化。
- 严格语义:接口明确拒绝思考参数时原样报错(不静默移除、不付费换字段重试);保存前有零费用的**映射预览**显示实际将发送/移除的字段。
- 对 DeepSeek 类默认开思考的模型,选"明确关闭"或低档可显著降低翻译延迟与费用;诊断记录与连通测试会显示返回的 reasoning_tokens 用量(注意:反代返回用量≠证明真实上游,插件不做上游身份声明)。
- 思考过程内容永不持久化、永不写入编辑器与日志。

## 0.2.8 更新重点（速度优化）

- **流式预览(默认开启)**:英文预览随生成逐步显示,不再等完整响应;写入 Claude 的最终文本仍只来自完整校验后的响应。Chat Completions 与 Responses API 均支持;网关不支持流式时自动回退,不产生额外请求。可在设置关闭。
- **会话级翻译缓存**:同一 Provider/模型/协议下,规范化后相同的原文即时回放上次结果(含纠错与警告),撤销输入、来回修改、恢复草稿不再重复计费。
- **等价变化不重译**:末尾敲空格/回车这类规范化后无变化的编辑,不再触发付费重译,也不再作废在途请求。
- 提速建议:翻译任务通常用 mini/flash 级小模型即可(速度常是大模型数倍);"关闭回译"档可把输出减半,延迟约减半。

## 0.2.7 更新重点（外部审计核验修复）

- **审阅门禁**：模型报告的纠错、歧义或数量/复述类警告存在时,自动同步暂停,英文保留在预览;由你核对后点"同步到 Claude"作为显式批准。
- 纠错语义边界扩展：极性/方向/状态单字反转（是↔否、开↔关、多↔少等）与大写/异体中文数字（壹贰…廿卅）一律本地拒绝。
- 中文数字数值核对：三十 → "fifty" 这类不一致现在会产生数量警告并进入审阅门禁（两字以上的中文数词）。
- 大量未翻译中文的输出（如「你好 a」）从警告升级为硬失败;回译中文比例过低同样硬失败。
- 暂停原因隔离：目标丢失自动恢复只会解除"目标丢失"本身造成的暂停,不再可能误清外部配置审查等其他暂停。
- 跨窗口保存回滚防护:回滚不再可能覆盖另一窗口保存中途写入的新 Key。
- Responses/Chat 解析严格化:只接受 completed 状态与 assistant 角色,finish_reason 白名单,对象标记校验,移除 legacy `text` 回退;`/models` 非模型列表结构显式报错。
- 更多密钥样式请求头（X-Access-Key、X-Auth、X-Signature 等）禁止存入普通额外 Header;诊断端点路径默认脱敏。
- 附件失败关闭扩展到仅带无障碍标签的兄弟节点芯片;写入命令执行前重新校验输入框可见可编辑;编辑器候选周期性重排名。
- Markdown 变长围栏（````…````）与多反引号行内代码、`~/`、`$VAR/`、`路径：/x`、`path=/x` 等形式纳入保护;保护词先去重再限额并支持重叠出现;占位符还原后超过 100,000 字符硬失败。

## 0.2.6 更新重点

- 新增 **OpenAI Responses API（`/v1/responses`）支持**：在设置的"接口协议"中显式选择;插件不自动猜路由、不跨协议重试。Responses 请求默认 `store: false`,不留存服务端响应。
- "立即翻译"按钮在暂停、换绑、清空草稿或后台重启中止翻译后不再可能永久禁用（0.2.5 引入的回归）。
- Claude 空态输入框经属性级 hydration 出现时可自动定位,不再需要先往页面输入内容。
- 普通换会话不再强制暂停、不再中止在途翻译;只有插件已写入的文本被孤立时才暂停。
- 页面刷新 / 后台服务重启导致的可恢复断连,在自动重绑成功后自动恢复,不再每次要求点"恢复"。

## 0.2.5 修复重点

- 中文数字纠错（如 三十→五十）现在会被本地拒绝并触发字面重译,不再能静默改变数量。
- 发送确认状态机修复两处：编辑器卸载且替换未挂载时真实发送仍归档；尾部换行式 Enter 不再遗留可被伪造成"已发送"的意图。
- 换绑失败时服务工作线程同步释放旧租约,旧标签页事件不再穿透到已解绑的面板。
- 暂停→恢复不再对未变化草稿重复计费翻译,也不再覆盖已核对的英文。
- 旧版本存储的非法 Base URL 不再永久卡死"保存设置"。
- 运行时兼容降级只随显式连通测试持久化,普通保存不再固化降级。

## 0.2.4 修复重点

- HTTP 200 但返回 HTML、空正文、普通文本、Responses API 或未知 JSON 结构时，不再误报 `empty_response`，而是显示可判定的协议错误与安全诊断元数据。
- 当请求落到裸 `/chat/completions` 或 `/models` 并收到网站 HTML 时，会提示 Base URL 很可能缺少 API 前缀（常见为 `/v1`）；扩展不会自行猜路由重试，避免重复计费。
- Claude 输入框定位会读取编辑节点父级语义，允许空输入状态下 disabled 的发送按钮作为关联证据，并支持较短的 Lexical/ProseMirror 内部编辑节点。
- 手动绑定会等待页面重渲染、重新寻找被 React/Lexical 替换的节点，并且只有在节点仍存在、可见、可编辑时才报告成功。
- Provider 诊断只保留状态码、Content-Type、响应大小、请求 ID、端点路径和结构摘要，不保留 Key、提示词、请求正文或完整响应正文。

## 0.2.2 修复重点

本版是在 v0.2.1 页面安全修复之上的**配置一致性与多窗口竞态修复版**。默认仍由同一次模型请求同时生成英文和中文回译。

最重要的变化：

- API Key 现在与**精确 Provider 目标和凭据代次**绑定；旧窗口不能把新 Key 配到旧 Base URL、旧鉴权方式或旧路由 Header 上。
- 另一个 Edge 窗口修改 Provider、模型、Key 或密钥保存方式时，当前窗口会作废在途请求并自动暂停，要求重新确认配置。
- 普通行为设置只保存行为设置，不再顺带重写旧 Provider 快照。
- 网关兼容能力降级默认只在当前运行上下文生效；只有用户明确执行连接测试并保存时才持久化。
- 自定义 Header 名称按大小写不敏感去重，避免同时出现 `X-Project` 与 `x-project` 等语义重复项。
- 网关若先拒绝 `response_format`、再拒绝 `temperature`，现在可以进行最多两步、逐项且有界的兼容降级；无关 400/422 不自动重试。
- Claude 编辑器定位新增 `contenteditable="plaintext-only"` 支持。
- 新增跨窗口存储事件回归：当前窗口自己的保存不会误暂停，外部 Provider/Key 变化一定会使旧请求失效。
- 删除面向运行时的“无绑定读取 Key”接口；翻译、模型检测和连接测试都必须携带并验证当前 Provider 身份。

## 核心行为

默认主请求要求模型返回：

```json
{
  "english": "Please review this code without rewriting its overall structure.",
  "back_translation": "请检查这段代码，但不要重写其整体结构。",
  "corrections": [],
  "ambiguous": []
}
```

回译有三档：

- **同一次请求同时翻译和回译（默认）**：一次调用，速度快、成本低；属于同一模型自检。
- **第二次独立请求回译**：第二个请求只收到英文，不看到中文原稿；核对更独立。
- **关闭回译**：只生成英文。

## 安全边界

- 中文输入只发生在 Edge 扩展侧边栏，不进入 Claude 页面 DOM。
- IME composition 期间不翻译、不写入。
- 英文完整生成并通过校验后才整段写入，不逐 token 修改 Claude 输入框。
- 中文原稿永不被插件静默改写；模型声称的明显错字会显式展示。
- 用户直接修改 Claude 中的英文后，自动覆盖立即暂停。
- 不自动点击发送、不模拟回车发送、不读取 Claude 回复、不抓取聊天历史、不调用 Claude 内部接口。
- 不做指纹伪装、随机节奏伪装或风控规避。
- 存在附件、图片、文件卡片、mention 或不可安全替换的富文本节点时，自动写入失败关闭并保留预览。

## 从 v0.2.x 升级

为了尽量保留扩展 ID、Provider、模型和 Key：

1. 关闭扩展侧边栏。
2. 备份原解压目录。
3. 用当前版本文件覆盖原来的**同一个解压目录**。
4. 在 `edge://extensions` 找到原扩展并点击“重新加载”。
5. 刷新已打开的 Claude 标签页。
6. 打开侧边栏，进入设置并点击一次“保存配置”。这会把旧的裸 Key 迁移为带 Provider 绑定和凭据代次的安全记录。
7. 运行一次“真实连通测试”和“页面诊断”。

若将新版本解压到另一个目录并重新“加载解压缩的扩展”，Edge 可能将其识别为另一扩展，旧存储不会自动继承。

### 旧 Key 迁移说明

- v0.2.1 及更早版本保存的裸字符串 Key，在 Provider 未变化时仍可临时读取，避免升级后立即失效。
- 下一次明确保存配置时，插件会生成新的 `credentialId`，将 Key 与规范化后的 Base URL、鉴权 Header、鉴权前缀和路由 Header 绑定。
- 一旦进入新记录格式，旧窗口或旧代码不能仅凭“同一个存储槽位”读取该 Key。

## 新安装

1. 解压 ZIP。
2. 在 Edge 打开 `edge://extensions`。
3. 开启“开发人员模式”。
4. 点击“加载解压缩的扩展”。
5. 选择直接包含 `manifest.json` 的解压目录（目录名任意，勿在版本间更换目录）。
6. 打开或刷新 `https://claude.ai/`。
7. 点击扩展图标打开侧边栏；未自动绑定时点击“绑定当前页”。

## Provider 配置

1. 打开“设置”。
2. 选择 OpenAI、xAI/Grok 或自定义模板。
3. 填写 Base URL 和 API Key。
4. 点击“检测模型”；若 `/models` 不可用，可直接手工填写模型 ID。
5. 选择主翻译模型和回译模式。
6. 独立回译模式下，另选回译模型。
7. 点击“真实连通测试”。
8. 保存；Edge 只为该 Provider 的精确 Origin 请求权限。

常用 Base URL：

```text
OpenAI: https://api.openai.com/v1
xAI:    https://api.x.ai/v1
```

自定义端点应兼容（按所选"接口协议"二选一）：

```text
GET  {baseUrl}/models             # 可选
POST {baseUrl}/chat/completions   # 接口协议 = Chat Completions（默认）
POST {baseUrl}/responses          # 接口协议 = Responses API
```

若供应商只提供 OpenAI Responses API（`/v1/responses`），在设置的"接口协议"里选择 Responses API。协议是显式配置：返回结构与所选协议不符时插件会明确报错并提示切换，不会自行改路由重试。Responses 请求携带 `store: false`，不在服务端留存响应内容。

Base URL 必须包含供应商要求的 API 前缀。例如供应商的实际端点是
`https://example.com/v1/chat/completions`，应填写 `https://example.com/v1`；也可粘贴完整的 `/chat/completions` 地址，保存时会规范化为 Base URL。不要只填写网站首页域名。

### 自定义 Header

- Header 名称按大小写不敏感去重。
- 不能设置 Cookie、Host、Origin、Referer、`Sec-*`、`Proxy-*`、`Content-Type`、`Accept` 或危险对象键。
- 普通额外 Header 中不能保存疑似 API Key、Token 或 Secret。
- 自定义鉴权 Header 与普通额外 Header 不能重名。

## 日常使用

- 在“中文原稿”输入内容，停顿后整稿翻译。
- `Ctrl+Enter`：翻译并同步，并取消尚未触发的防抖任务。
- `Esc`：仅暂停自动翻译与同步（恢复请点"恢复"按钮）；IME 候选状态下不会误触。
- 超过设置中的实时翻译阈值后，改为点击“翻译整段并同步”。
- 超过 50,000 字符的草稿会保留，但必须拆分后翻译。
- 英文写入失败、目标不在前台、存在附件或需要未授权的焦点写入时，只保留预览/复制模式。
- 最终发送始终由你点击 Claude 自己的发送按钮。
- 普通 Enter 若插入换行，不会被记录为发送；只有可信发送意图后输入框清空，草稿才归档。
- 活动草稿、历史和诊断按 Edge 窗口隔离。
- 其他窗口更改 Provider 或 Key 时，当前窗口作废在途请求并直接采纳新配置继续；只有读取失败才会暂停。

## 回译选择

### 同请求模式

适合日常短提示词。它能帮助发现明显漏词、语气变化和结构问题，但同一个模型看到了中文原稿，可能沿用同一理解偏差。

### 独立模式

适合重要提示词。第二次请求只收到英文，不包含中文原稿。它仍不是形式化证明，但比同请求自检更独立。

### 关闭回译

适合已经人工检查英文，或希望把请求量降到最低的场景。

## 明显错字规则

只有同时满足以下边界，模型提出的纠错才可能被采纳：

- 原片段确实存在且在原稿中只有一个位置；
- 只涉及很短的汉字片段；
- 改动核心不超过 1–2 个汉字；
- 不涉及数字、英文、代码、路径、URL、产品名、否定、礼貌或情态强度词；
- 每篇短稿最多两处；
- 上下文中的预期读法近乎唯一。

越界纠错会被本地拒绝，并自动按“完全禁止纠错”重译一次。中文原稿不会被修改。

## 首次页面验证：必须完成

Claude 的富文本编辑器可能随时更新。首次安装、升级或 Claude 明显改版后，请在临时新对话中打开“页面诊断”：

1. 确认“输入框：已定位”。
2. 点击“写入诊断短句（不发送）”。
3. 确认 Claude 输入框中文字完整正确。
4. **由你人工点击 Claude 发送按钮**。
5. 核对实际发出的消息是否与发送前可见文字逐字一致。
6. 再测试多段、连续空行、Markdown、代码块和附件。

任何一项出现“看见的是新英文，实际发出的是旧内容”，立即关闭聚焦写入，只使用英文预览和复制模式。插件刻意不读取已发送消息，因此这一步无法在未登录你的 Claude 账户的构建环境中自动完成。

## 密钥与隐私

- `storage.local` 与 `storage.session` 均限制为 `TRUSTED_CONTEXTS`，内容脚本不能读取 Key。
- Key 可本地持久保存，也可仅保留当前 Edge 会话。
- Key 记录与 Provider 目标和凭据代次绑定；请求时必须同时验证当前存储中的 Provider、保存模式和 Key 记录。
- Key 不进入 Claude 页面、内容脚本、URL、诊断日志或配置导出。
- 切换 Key 存储模式时，新 Key、旧 Key 删除、Provider 与设置作为事务保存；失败会回滚。
- 外部窗口变更 Provider、Key 或其代次时，在途请求会作废并采纳新配置继续；旧请求不可能拿新 Key 配旧 Provider。
- 额外请求头不能保存 Cookie、Authorization、API Key、Token、Secret 或正文控制头。
- 本地持久 Key 不是服务器密钥库。建议使用单独、可撤销、可限额的 Key。
- 扩展无遥测，不读取 Claude 对话、回复、Cookie、浏览历史或其他标签页正文。

详见 [PRIVACY.md](PRIVACY.md)。

## 已知边界

- 支持 Chat Completions 与 Responses API 两种协议，但必须显式选择；返回结构与所选协议不符会明确报错，不会自动切换。Responses 模式忽略 `reasoning` 输出项，只接受纯 `output_text` 结果。
- Provider 已开始处理的请求即使客户端 Abort，也可能仍产生费用。
- 同请求回译是自检，不是独立语义证明。
- 自定义兼容网关差异很大；插件只对明确的不兼容参数进行最多两步、逐项且有界的降级。
- Claude 改版时，插件会优先失败关闭并退化为复制模式，而不是不断试探页面。
- 当前不支持需要 URL query 参数（例如特定 `api-version`）的 Provider Base URL。
- 真实 Claude 账户中的“输入框可见内容＝实际发送内容”必须由用户人工验证。
- 数量核对覆盖阿拉伯数字与两字以上中文基数词的数值一致性；单字数词（三个、一次）、语义化数词短语（一点、几个）、正负号与上下文角色互换超出本地校验能力，命中数量警告时请人工核对后再同步。
- 通用相对路径（如 `src/index.js`）不自动保护——同形的中文写法（如 他/她）会被误锁定；需要原样保留时请加入“保护词”。
- 两个 Edge 窗口同时首次打开旧版本草稿时，迁移可能在两个窗口各留一份副本；无数据丢失，可手动清理多余草稿。
